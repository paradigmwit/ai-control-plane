# ADR-003: Policy Engine Layer

## Status
Approved

## Context
Uncontrolled execution of tools can cause:

- Security breaches
- Sensitive data leaks
- Cost overruns
- Violations of enterprise rules

## Decision
Introduce a **Policy Engine** that validates each step in a plan before execution:

- Permission checks per tool
- Step-level execution limits
- Token/API cost ceilings
- Safety constraints (e.g., no external emails without approval)

Policy engine runs **after plan validation and before execution**.

## Alternatives Considered
1. No policy enforcement
    - Pros: Simple
    - Cons: Unsafe, non-compliant, unpredictable
2. Policy embedded in Planner
    - Pros: Early filtering
    - Cons: Hard to maintain, not enforceable at runtime

## Consequences
- Controlled execution
- Safety and compliance
- Clear audit trail
- Centralized policy management
