# ADR-001: Separation of Planner and Executor

## Status
Approved

## Context
Large Language Models (LLMs) are powerful but non-deterministic. Directly coupling LLM reasoning with execution introduces risks:

- Hallucinated tool calls
- Non-reproducible results
- Hard-to-debug failures
- Security and policy violations

## Decision
We will **separate the planner (LLM reasoning) from the execution engine (deterministic DAG execution)**.

- **Planner**: Generates a structured plan (DAG) from user input. Responsible only for reasoning.
- **Executor**: Deterministically executes validated steps. Handles retries, fallback, and parallelism.

All execution must go through the execution engine; the LLM never directly executes business logic.

## Alternatives Considered
1. **LLM executes steps directly**
    - Pros: Simple implementation
    - Cons: Hallucinations, unpredictable execution, no policy enforcement
2. **LLM generates partial execution code**
    - Pros: Some determinism
    - Cons: Still requires trust in model output; debugging difficult

## Consequences
- Clear separation of reasoning vs execution
- Execution engine can enforce policy, cost, and observability
- Plan can be validated and replayed
- Easier to test and maintain
