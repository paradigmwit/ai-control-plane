# ADR-007: Cost-Aware Execution & Token Management

## Status
Approved

## Context
LLM-driven workflows and external API calls incur cost. Without proactive management:

- Token consumption may exceed budget
- High-frequency API calls may trigger rate limits or financial impact
- Unchecked workflows may execute unnecessary steps

Cost management must be integrated into planning and execution.

---

## Decision
Introduce **Cost-Aware Execution**:

1. **Planner Cost Estimation**
    - Planner estimates token usage and tool cost per step
    - Optional pre-execution scoring to prune expensive plans

2. **Execution-Level Cost Tracking**
    - Execution Engine tracks token/API consumption in real time
    - Updates per step for reporting

3. **Cost Enforcement**
    - Policy engine can enforce maximum allowed cost per plan
    - Steps exceeding thresholds may be skipped, batched, or modified
    - Alerts are generated for cost violations

4. **Integration with Observability**
    - Logs per-step token/API usage
    - Provides metrics for evaluation and optimization

---

## Alternatives Considered

1. **No cost tracking**
    - Pros: Simple
    - Cons: High risk of budget overruns

2. **Planner-only cost estimation**
    - Pros: Early pruning
    - Cons: Cannot account for retries or runtime variations

3. **Post-execution cost auditing**
    - Pros: Easy to implement
    - Cons: Reactive; doesn’t prevent over-budget execution

---

## Consequences

- Workflow execution remains within cost constraints
- Token and API usage is predictable and auditable
- Enables iterative cost optimization
- Supports enterprise budgeting and governance
