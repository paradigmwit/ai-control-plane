# ADR-002: JSON-Only Execution Plan

## Status
Approved

## Context
We need the planner to generate plans in a way that is machine-readable, deterministic, and enforceable. Allowing free-text plans risks hallucinations and inconsistent schemas.

## Decision
All plans from the LLM **must be strictly JSON conforming to an ExecutionPlan schema**:

- Execution steps with IDs, tool names, input objects, and dependency lists
- Confidence scores per step
- Optional metadata (cost estimates, retry limits)

## Alternatives Considered
1. Free-text plans
    - Pros: Easier for LLM
    - Cons: Hard to validate, parse, enforce
2. Mixed text + JSON
    - Pros: Human-readable
    - Cons: Parsing complexity; higher chance of errors

## Consequences
- Enables plan validation before execution
- Supports policy enforcement
- Execution engine can deterministically resolve DAG
- Reproducible execution traces
